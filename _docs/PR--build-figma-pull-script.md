# PR: `build/figma-pull-script` → `main`

### What this does

Adds `scripts/figma-pull.mjs` — a Figma REST API pull script that reads variables
from the CGDC-DS Figma file and writes them into `tokens/tokens.json`.

Also updates `README.md` with the current accurate pipeline architecture and a
documented architectural decision about Style Dictionary.

### Why

Closes the open half of the token pipeline. Previously, color and semantic token
changes made in Figma Variables had to be manually transcribed into `tokens.json`.
The pull script automates that direction: `FIGMA_TOKEN=pat npm run tokens:pull`
fetches all pullable tokens from Figma and merges them into `tokens.json`, after
which `npm run tokens:build` regenerates the SCSS.

---

### What the script pulls

From the Figma Variables API (`GET /v1/files/:key/variables/local`):

| Collection | What's pulled |
|---|---|
| primitives | `color/*` only — all three palettes (primary, secondary, neutral) |
| semantic | `space/*`, `radius/*`, `color/*`, `type/*` |
| component | `mosaic/*`, `space/content-rhythm/*` |

### What the script deliberately skips

| Skipped | Reason |
|---|---|
| `primitives.scale.*` | Math expressions (`calc(var(--scale-base) * 1.75)`) must be preserved in CSS output for the 2-col grid `--scale-base: 14px` override to work at runtime. Figma stores resolved values (28px) — pulling these would break the cascade. |
| `primitives.font-family.*` | Stable string values, hand-authored. Change rarely, no benefit to pulling. |
| `primitives.font-weight.*` | Stable numeric values, hand-authored. |
| `primitives.font-case.*` | Hand-authored. |
| `primitives.text.*` (tracking) | Hand-authored. |
| `component.layout.*` | Layout geometry constants (`fieldWidth: 1896`), not design tokens. |

### Alias resolution

VARIABLE_ALIAS values are resolved to DTCG reference strings.
`{ type: 'VARIABLE_ALIAS', id: 'VariableID:2749:1054' }` → `{scale.25}`
This matches the existing reference format in `tokens.json`.

### Merge behavior

The script reads the existing `tokens.json` before writing, merges at the leaf
level, and preserves hand-authored content. It does NOT overwrite the entire file.
`$description` fields on existing tokens are preserved. `$themes` and `$metadata`
at the root level are preserved.

### What to verify WHEN running the pull (not part of this commit check)

When a developer runs `FIGMA_TOKEN=pat npm run tokens:pull`:
- Script outputs count of pulled and skipped tokens
- `tokens.json` updates in `primitives.color`, `semantic`, and `component.mosaic`
- `primitives.scale` is unchanged
- `npm run tokens:build` succeeds after the pull
- `_tokens--primitives.scss` still contains `calc(var(--scale-base) * ...)` expressions

### What this does NOT change

- The build pipeline — `npm run tokens:build` is unchanged
- `tokens.json` structure — the pull script merges, never replaces
- Any SCSS files — those are regenerated by `tokens:build`, not this script
- Any Eleventy templates or YAML data files
- The design system hub token viewer
